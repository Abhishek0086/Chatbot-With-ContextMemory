How context memory works

LLMs are stateless — they have no built-in memory. We implement it by maintaining conversation history and replaying it with every request.

01
User sends message
The message is appended to the in-memory session history stored on the Flask server.
02
History is replayed
The full conversation history (all prior turns) is sent with each new API call to Claude.
03
Model responds
Claude reads all previous messages as context and generates a response that accounts for the entire conversation.
04
Reply stored
The assistant's response is appended to history, growing the context for the next turn.
