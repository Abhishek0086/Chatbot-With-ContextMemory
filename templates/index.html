How context
memory works

LLMs are stateless — no built-in memory. We implement it by maintaining history and replaying every turn with each request.

01

User sends message

Appended to the in-memory session store on the Flask server alongside a timestamp.

02

Full history replayed

All prior turns — user and assistant — are sent together with each new API call to Claude.

03

Model reads everything

Claude sees the complete conversation and generates a response informed by every previous turn.

04

Reply stored, context grows

The assistant's response is appended to history, making the next call even richer in context.

app.py
# context memory = full history on every call
 
@app.route("/api/chat", methods=["POST"])
def chat():
  session_id = data.get("session_id")
  msg = data.get("message")
 
  # append user turn to history
  sessions[session_id].append({
    "role": "user",
    "content": msg
  })
 
  # THIS is context memory ↓
  response = client.messages.create(
    model="claude-sonnet-4-20250514",
    messages=sessions[session_id]
  )
